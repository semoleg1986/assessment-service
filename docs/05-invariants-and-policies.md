# Инварианты И Политики

## Инварианты модели

1. В тесте должен быть хотя бы один вопрос.
2. `grade` теста ограничен диапазоном `1..4`.
3. У одного назначения не может быть больше одной активной попытки.
4. После `submit` ответы попытки неизменяемы.
5. Расчет `score` детерминированный: сумма `awarded_score`.
6. Для `question_type=text`:
   - обязателен `answer_key`
   - запрещены `options` и `correct_option_id`
7. Для `question_type=single_choice`:
   - минимум 2 варианта ответа
   - уникальные `option_id` и `position` в рамках вопроса
   - `correct_option_id` обязан указывать существующий вариант
   - у корректного варианта `diagnostic_tag = null`
8. `MicroSkillNode.topic_code` должен ссылаться на существующую тему того же `subject_code` и `grade`.
9. Граф `predecessor_ids` ацикличен.

## Политики проверки ответа

- `text`:
  - корректность: точное сравнение с `answer_key` (после `strip`)
  - при ошибке может выставляться `resolved_diagnostic_tag` по первому совпавшему `text_distractor`
- `single_choice`:
  - корректность: `selected_option_id == correct_option_id`
  - при ошибке в `resolved_diagnostic_tag` записывается тег выбранной опции (если задан)

## Политики доступа

- Родитель работает только со своими детьми (через `user-web`/BFF).
- Ребенок запускает и отправляет только свои назначения.
- Админ/контент-менеджер создают контент, тесты и читают диагностику.

## Политика import-контракта (`v1.2`)

- Валидация и применение поддерживают `error_mode`:
  - `collect`: собрать все найденные ошибки
  - `fail_fast`: остановиться на первой ошибке
- Для `tests/questions` включена entity-level isolation:
  - невалидные сущности пропускаются
  - валидные применяются
  - статус: `validated_with_errors` или `completed_with_errors`
- Идемпотентность: повторный apply без изменений должен вернуть `imported=0`.

## Политика сигналов (MVP, rule-based)

- `norm`: >= 85%
- `risk`: 70-84%
- `gap`: 50-69%
- `critical_gap`: < 50%

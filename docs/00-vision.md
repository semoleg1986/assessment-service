# Видение

Assessment Service — основной контекст оценки учебных достижений младших школьников.
Сервис хранит тесты, назначения, попытки и ответы, рассчитывает результат по правилам и формирует сигналы пробелов.

## Цели
- Рано и точно выявлять пробелы в знаниях.
- Показывать результат по предметам, темам и микро-умениям.
- Быстро переводить диагностику в коррекционный план.
- Поддерживать единый подход для 1-4 классов.

## Не входит в зону ответственности
- Аутентификация и выпуск токенов (Auth Service).
- UI-оркестрация и BFF-логика.
- Доставка уведомлений (Notification Service).

## Принципы
- Сначала rule-based MVP, затем расширение.
- Инварианты реализуются в доменном слое.
- Явные API и версионируемые контракты.
- Прозрачные переходы состояний назначений и попыток.
